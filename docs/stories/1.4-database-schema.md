## Technical Notes

### CRITICAL: Same Supabase Instance
PUBLYQ uses the **same Supabase instance** as OPB Crew. Tables must NOT conflict with existing ones.

Existing OPB Crew tables to NOT touch:
- `user_profiles` (already exists — REUSE, add new columns if needed)
- `generated_content` (OPB Crew — do NOT modify)
- `waitlist` (already exists — REUSE)

### New Tables
- `brand_profiles` — NEW
- `voice_profiles` — NEW (replaces `user_voice_profiles` from OPB Crew)
- `generated_carousels` — NEW
- `rate_limits` — may already exist from migration 010
- `audit_log` — may already exist from migration 010

### SQL Safety Rules (CRITICAL)
- `user_profiles` already exists → use `ALTER TABLE ADD COLUMN IF NOT EXISTS` for new columns
- `rate_limits` and `audit_log` may exist from migration 010 → use `CREATE TABLE IF NOT EXISTS`
- All new tables (`brand_profiles`, `voice_profiles`, `generated_carousels`) → use `CREATE TABLE IF NOT EXISTS`
- `waitlist` already exists → do NOT recreate, only verify
