# Story 1.3 — Setup Clerk Auth (Sign-in, Sign-up, Middleware)

> Epic: PUBLYQ MVP
> Priority: P0 | Points: 2
> Dependencies: Story 1.1
> Assigned to: @dev

---

## User Story

**As a** user,
**I want to** sign up and sign in to PUBLYQ using email or Google,
**So that** my data is protected and my progress is saved.

---

## Acceptance Criteria

- [x] Clerk installed and configured (`@clerk/nextjs`)
- [x] `/sign-in` page renders Clerk SignIn component
- [x] `/sign-up` page renders Clerk SignUp component
- [x] `proxy.ts` protects all `/dashboard/*` routes (redirect to `/sign-in` if not authenticated) — Next.js 16 uses proxy.ts instead of middleware.ts
- [x] `proxy.ts` checks beta access via `BETA_USERS` array (same as OPB Crew)
- [x] Unauthorized users redirected to `/#waitlist`
- [x] Clerk env vars documented in `.env.local.example`
- [x] Auth pages styled to match PUBLYQ brand (dark theme, Space Grotesk)

- Secret key: `CLERK_SECRET_KEY`
- Sign-in URL: `/sign-in`
- Sign-up URL: `/sign-up`
