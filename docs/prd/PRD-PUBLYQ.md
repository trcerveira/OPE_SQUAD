## 1. Executive Summary

**Product:** PUBLYQ (publyq.ai)
**Tagline:** "Cria o conteúdo de uma semana numa manhã."
**Category:** AI-powered Content Creation SaaS
**Market:** Portugal (first) → Global (later)
**Language:** Português
**Target Launch:** MVP amanhã (2026-03-13) para teste com 5 pessoas
**Validation Gate:** 5 pessoas a usar grátis + feedback real (not vanity waitlist)

PUBLYQ é uma plataforma SaaS que transforma uma manhã por semana em todo o conteúdo para Instagram — na voz autêntica do utilizador, não na de um robô. Combina Brand DNA, Voice DNA e um Kaizen Loop proprietário que melhora automaticamente a qualidade do conteúdo todas as semanas.

### O Problema Real
As pessoas não publicam. Não têm consistência. E é a consistência que faz contas e negócios crescerem. Exemplo real: o fundador tem uma empresa que fatura mais de 1M€ e não tem redes sociais.

### Why Now
- Creator economy: $205B (2024) → $480B (2027)
- 207M+ creators globally, 58.7% adopting AI tools
- Portugal: 62% de adopção de genAI (mais alto na Europa)
- Zero competitors integrate batch creation + deep voice cloning + automated weekly improvement
- Proof of concept: AutoShorts.ai reached $97K MRR with a solo founder

### The Moat
**Kaizen Loop + Toyota Way** — O que a Toyota fez para os carros, o PUBLYQ faz para a criação de conteúdo. Todas as semanas, o sistema analisa o que funcionou, aprende com os dados de engagement e ajusta o próximo ciclo automaticamente. Semana 12 é dramaticamente melhor que semana 1. Zero concorrentes têm isto.

O moat real não é o código (podem copiar o front-end) — é o sistema por trás, a zona de genialidade do fundador e a visão integrada. "Nada se cria, tudo se transforma."

## 2. Problem Statement

### Who Has This Problem
PMEs portuguesas (1-5 pessoas) sem equipa de marketing que:
- Sabem que precisam de publicar conteúdo consistentemente mas não conseguem manter
- Não têm ideias do que publicar ou são perfeccionistas
- Quando usam IA, o output é genérico — "parece o ChatGPT"
- Publicam quando se lembram — sem plano, sem consistência
- Sabem que "se não estás na internet, não existes"

**Também serve para:** content creators, pessoas a começar do zero, qualquer negócio sem equipa de marketing dedicada.

### The Core Problem
> "Não publico. Não tenho tempo, não tenho ideias, e quando uso IA parece tudo igual. O meu negócio cresce por boca-a-boca mas sei que devia estar nas redes. Tenho uma empresa que fatura mais de 1M e não tenho redes sociais."

### O Custo de Não Publicar
- Uma equipa de social media custa no mínimo 1 pessoa = 1 ordenado mínimo = €1.400/mês
- Sem presença online = invisível para novos clientes
- Inconsistência = algoritmo penaliza = menos alcance = mais invisibilidade

### Current Solutions (and why they fail)

| Solução | Porque Falha |
|---------|-------------|
| Fazer manualmente | Insustentável — cansa em 2 semanas |
| Contratar pessoa de marketing | Demasiado caro para PMEs (mín. €1.400/mês) |
| Usar ChatGPT/Claude directo | Output genérico, sem voz, sem sistema, sem melhoria |
| Jasper / Copy.ai | Foco num post de cada vez, sem batch, sem Kaizen, preço enterprise |
| Typefully / Taplio | Só LinkedIn, sem voice DNA |
| Buffer / Publer | Ferramentas de agendamento, não de criação |
| Lately.ai | Mais próximo mas focado em enterprise ($99-239/mês) |

### Validated Whitespace
Nenhuma ferramenta combina os quatro:
1. Captura profunda de voz (não "tom: profissional" mas voice DNA real)
2. Criação batch (conteúdo da semana inteira numa manhã)
3. Adaptação multi-plataforma (mesma mensagem, formato nativo por plataforma)
4. Loop de melhoria automático (Kaizen — aprende o que funciona)

## 4. Content Machine Method™ — The Core Framework

### The 4 Layers

```
LAYER 1: FOUNDATION (one-time setup, ~30 min)
  ├── Brand DNA — who you are, what you stand for
  ├── Voice DNA — how you talk, write, think
  ├── Audience DNA — who you serve, what they need
  └── Output: Complete voice profile the AI learns ONCE

LAYER 2: MACHINE (weekly, uma manhã)
  ├── Batch Day — 1 sessão por semana (manhã)
  ├── Input: topic/theme + voice profile
  ├── AI generates: 7+ posts para Instagram (MVP)
  ├── Formats MVP: carousels Instagram
  └── Output: Conteúdo da semana pronto a publicar

LAYER 3: KAIZEN (automated, weekly)
  ├── System analyzes: what performed, what didn't
  ├── Identifies: patterns, hooks, topics that resonate
  ├── Adjusts: next week's content strategy automatically
  ├── Reports: weekly improvement metrics
  └── Output: Each week is better than the last

LAYER 4: SCALE (when ready)
  ├── Multi-platform: Instagram, LinkedIn, X, Email, YouTube
  ├── Multi-format: text, carousel, video script, email
  ├── One input → content adapted for every channel
  └── Output: Omnipresence without extra work
```

### Why This Order Matters
- Foundation MUST come first — without voice DNA, all content is generic
- Machine NEEDS Foundation — batch creation uses the voice profile
- Kaizen NEEDS Machine — can't improve what you don't measure
- Scale NEEDS Kaizen — expanding garbage at scale is still garbage

### Veto Conditions (Content Engine)
- Content WITHOUT Voice DNA → BLOCKED
- Post WITHOUT call-to-action → BLOCKED
- Wrong format for platform → BLOCKED
- Publish without user approval (V1) → BLOCKED
- Skip Foundation layer → BLOCKED

> "Mais vale ser muito bom em uma do que normal em várias." — Telmo

#### F1: Brand DNA
- **O quê:** Setup guiado que captura a identidade da marca/pessoa
- **Como:** Perguntas sobre missão, valores, posicionamento, personalidade
- **Output:** Brand DNA profile completo guardado na base de dados
- **Código base:** Reaproveitar do OPB Crew (já existe) + melhorar
- **Acceptance Criteria:**
  - [ ] Utilizador completa Brand DNA em menos de 15 minutos
  - [ ] Sistema guarda perfil estruturado
  - [ ] Perfil é usado como input para geração de conteúdo
  - [ ] Utilizador pode editar/actualizar a qualquer momento

#### F2: Voice DNA
- **O quê:** Captura profunda da voz e tom do utilizador
- **Como:** Perguntas sobre nicho, oferta, dor, tom, diferencial + exemplos de escrita
- **Output:** Voice DNA profile que garante conteúdo único (não parece ChatGPT)
- **Código base:** Reaproveitar do OPB Crew (já existe) + melhorar
- **Acceptance Criteria:**
  - [ ] Utilizador completa Voice DNA em menos de 10 minutos
  - [ ] Sistema guarda perfil de voz estruturado
  - [ ] Conteúdo gerado soa como o utilizador (blind test)
  - [ ] Voice DNA é obrigatório antes de gerar conteúdo

#### F3: Carrossel Instagram
- **O quê:** Gerar carrosséis para Instagram com Brand DNA + Voice DNA
- **Como:** Utilizador dá tema → IA gera carrossel com slides + texto + CTA
- **Output:** Carrossel pronto a publicar no Instagram
- **Código base:** Reaproveitar Design Machine do OPB Crew + melhorar
- **Acceptance Criteria:**
  - [ ] Gera carrossel com 5-10 slides
  - [ ] Texto usa Voice DNA (não genérico)
  - [ ] Cores da marca aplicadas automaticamente
  - [ ] CTA no último slide
  - [ ] Export como imagens

## 7. Technical Architecture

### Stack

| Layer | Technology | Rationale |
|-------|-----------|-----------|
| Frontend | Next.js 16 + TypeScript + Tailwind | SSR, performance, developer experience |
| Auth | Clerk | Fast setup, social login, webhook sync |
| Database | Supabase (PostgreSQL) | Real-time, RLS, auth integration |
| AI Engine | Claude API (Anthropic) | Best at voice matching, long context |
| Deploy | Vercel | Next.js native, edge functions, preview deploys |
| Payments | Stripe | Industry standard, trial support |
| Affiliates | Rewardful | Stripe-native, automatic tracking |
| Domain | publyq.ai | Live, DNS → Vercel |

### Database Schema (Core Tables)

```sql
-- Users (synced from Clerk)
user_profiles (
  id UUID PK,
  clerk_id TEXT UNIQUE,     -- user_xxxxxxx
  email TEXT,
  name TEXT,
  plan TEXT DEFAULT 'free',  -- free, trial, starter, pro
  onboarding_step TEXT,      -- foundation, voice_dna, ready
  created_at TIMESTAMPTZ
)

-- Voice DNA (one per user)
voice_profiles (
  id UUID PK,
  user_id TEXT UNIQUE,            -- clerk_id
  answers JSONB,                  -- 8 questions (raw user input)
  voice_dna JSONB,                -- AI-generated profile {arquetipo, tom, vocabulario, regras}
  version INT DEFAULT 1,
  created_at TIMESTAMPTZ,
  updated_at TIMESTAMPTZ
)

-- ⚠️ TABLES BELOW ARE PHASE 1+ (NOT MVP) — included for reference only

-- Generated Content
content_batches (
  id UUID PK,
  user_id TEXT,
  topic TEXT,
  batch_date DATE,
  status TEXT DEFAULT 'draft',  -- draft, approved, published
  kaizen_score FLOAT,           -- improvement metric
  created_at TIMESTAMPTZ
)

content_pieces (
  id UUID PK,
  batch_id UUID REFERENCES content_batches(id),
  user_id TEXT,
  platform TEXT,            -- instagram, linkedin, x, email
  format TEXT,              -- post, carousel, thread, email
  content JSONB,            -- {text, hashtags, cta, slides[]}
  scheduled_for TIMESTAMPTZ,
  published_at TIMESTAMPTZ,
  engagement JSONB,         -- {likes, comments, shares, clicks}
  status TEXT DEFAULT 'draft',
  deleted_at TIMESTAMPTZ    -- soft delete
)

-- Kaizen Loop Data
kaizen_reports (
  id UUID PK,
  user_id TEXT,
  week_number INT,
  analysis JSONB,           -- {top_content, patterns, recommendations}
  improvements_applied JSONB,
  score_delta FLOAT,        -- week-over-week improvement
  created_at TIMESTAMPTZ
)

-- Waitlist (already exists)
waitlist (
  id UUID PK,
  email TEXT UNIQUE,
  nome TEXT,
  created_at TIMESTAMPTZ
)
```

### API Routes

```
POST   /api/voice-dna          → Create/update voice profile
GET    /api/voice-dna          → Get user's voice profile

POST   /api/generate           → Generate content batch
GET    /api/batches             → List user's batches
GET    /api/batches/:id         → Get batch with content pieces
PATCH  /api/content/:id         → Edit content piece
POST   /api/content/:id/approve → Approve for publishing

POST   /api/kaizen/analyze      → Trigger weekly analysis
GET    /api/kaizen/reports       → Get improvement reports

POST   /api/waitlist            → Add to waitlist (exists)
POST   /api/checkout            → Create Stripe checkout session
POST   /api/webhooks/stripe     → Handle Stripe events
POST   /api/webhooks/clerk      → Sync Clerk user data
```

### Security Architecture
- Clerk handles all authentication
- Supabase RLS blocks direct access (USING(false))
- Service role key on server only — security at API route level
- Rate limiting on all AI endpoints (DB-based, works in serverless)
- Audit log for sensitive operations
- Zod validation on all inputs
